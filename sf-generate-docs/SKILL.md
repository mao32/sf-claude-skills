# Salesforce Documentation Generator

## Role
Generate two professional .docx documents per user story that import natively into Google Docs.

## Document 1: docs/TECH_<US-ID>.docx

**Audience:** Architect, tech lead, developers

**Sections:**
1. Header: "Technical Design Document", US-ID, date, "Generated by Claude Code"
2. Overview: user story summary + acceptance criteria
3. Data Model: tables of objects (Name, Type, Action) and fields (API Name, Type, Required, Description)
4. Architecture: patterns used, class table (Class → Type → Methods → Description), logic flow
5. Automation: flows, triggers/handlers, validation rules (object, condition, message)
6. Security: permission sets, field-level security considerations
7. Test Strategy: scenario table (# | Scenario | Type | Input | Expected | Test Class), coverage results
8. Deployment: package.xml contents, dependencies, post-deploy instructions
9. Risks & Notes: risks with mitigations, impacts, notes for architect

## Document 2: docs/FUNC_<US-ID>.docx

**Audience:** Business analyst, product owner, end users

**Sections:**
1. Header: "Documentazione Funzionale", US-ID, date
2. Context: business need (non-technical), benefits, user roles
3. Functional description: step-by-step from user perspective, concrete examples
4. Operational impact: what changes, new fields/objects and where to find them
5. Validation rules: user messages and how to resolve
6. Permissions: who can see/modify, how to request access
7. Functional test scenarios: table (Step | User Action | Expected Result)
8. FAQ: anticipated questions with answers

## DOCX Rules (for Google Docs compatibility)

- Use docx-js (npm docx): `const { Document, Packer, Paragraph, TextRun, Table, TableRow, TableCell, HeadingLevel, BorderStyle, WidthType, ShadingType, AlignmentType, LevelFormat } = require('docx');`
- Page: A4 (width: 11906, height: 16838 DXA)
- Font: Arial 12pt (size: 24 half-points)
- Heading1: 16pt bold (size: 32), Heading2: 14pt bold (size: 28)
- Margins: 1440 DXA (1 inch) all sides
- Tables: ALWAYS WidthType.DXA (NEVER PERCENTAGE — breaks Google Docs)
- Table width: 9026 DXA. columnWidths MUST sum to table width.
- Header row shading: fill "#D5E8F0", type ShadingType.CLEAR (NOT SOLID)
- Cell margins: { top: 80, bottom: 80, left: 120, right: 120 }
- Borders: BorderStyle.SINGLE, size 1, color "CCCCCC"
- NEVER \n — use separate Paragraphs
- NEVER unicode bullets — use LevelFormat.BULLET with numbering config
- Override heading styles with exact IDs "Heading1", "Heading2" + outlineLevel

## Output
Show: confirmation of files created, content summary, instruction "Drag .docx to Google Drive — opens directly as Google Docs"
